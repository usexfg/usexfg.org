v10 — new upgrades @ block 1M

Ring-Sig Commitment Deposits

Original CryptoNote deposits pioneered by projects like DigitalNote or Conceal, directly name the spending (deposit) input on the withdrawal transaction— making ring signatures mathematically impossible.
Fuego solves this privacy leak by utilizing cryptographic commitments for deposit transactions to create indistinguishable outputs, and enable XFG withdrawals & deposits to use a maximum-achievable ring signature set with OSPEAD decoy selection— by default, same as all Fuego transactions.
v10 also introduces Dandelion++ for IP-level network privacy, sub-addresses for wallets, 'fire' aliases, & more.
